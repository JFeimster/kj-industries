# Markdown Usage Guide

This repo uses Markdown for operational documentation, AI instructions, copy guidelines, and roadmap planning.

Markdown files are not currently rendered as public website pages.

## Use Markdown For

- AI coding assistant instructions
- roadmap planning
- compliance rules
- launch checklists
- copy guidelines
- future page specs
- QA notes
- partner program notes

## Use HTML For

- public website pages
- conversion pages
- embedded Tally forms
- SEO landing pages
- pages that Vercel should serve directly

## Current Markdown Files

```text
README.md
AGENTS.md
ROADMAP.md
COMPLIANCE.md
MARKDOWN-USAGE.md
```

## What About `agents.md`?

Use `AGENTS.md` for instructions to AI assistants and repo operators.

It should not be confused with a public `agents.html` page.

## What About `agents.html`?

An `agents.html` page would be public-facing. It could be used later for:

- funding agent recruitment
- broker resources
- partner program details
- agent onboarding
- sales enablement

For now, `partner.html` is the public partner recruitment page. Create `agents.html` only if K & J Industries needs a more explicit agent/broker recruiting page later.

## Recommended Workflow

When planning a new page:

1. Create the spec in Markdown.
2. Review copy and compliance language.
3. Build the public page in HTML.
4. Add internal links.
5. Test locally.
6. Deploy.

Markdown is the blueprint. HTML is the storefront.
